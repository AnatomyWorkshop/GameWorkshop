# TextSessionPage 交互细则

> 状态：MVP 专项文档（2026-04-11）
> 作用：只描述 Text 游玩页，不再承担目录页或详情页规划。

---

## 一、页面定位

`TextSessionPage` 是当前 MVP 的核心游玩界面。  
它只服务一件事：

> 让用户在一个稳定、低干扰的 Text 界面里完成持续游玩。

它不承担：

- 游戏详情解释
- 公共发现
- 个人库管理

这些分别交给其他三页。

---

## 二、固定布局

Text 游戏页固定采用三段式：

```
TopBar
中段消息流
底端输入区
```

不再使用“额外浮动工具层 + 边栏常驻按钮”的方案。

---

## 三、TopBar 结构

从左到右固定为：

1. **出口**
2. **标题**
3. **设定 / 世界书入口**
4. **统计 / 搜索入口**
5. **存档入口**
6. **其它 `···`**

### 3.1 出口

- 返回上一层
- 不承载更多功能

### 3.2 标题

- 显示当前游戏标题
- 始终居中优先

### 3.3 设定 / 世界书入口

打开同一侧边面板，用于：

- 游戏公开设定查看
- 世界书只读查看

它不是两个并排主按钮。

### 3.4 统计 / 搜索入口

打开同一侧边面板，用于：

- 当前会话统计摘要
- 会话内搜索（后续）

### 3.5 存档入口

打开当前游戏相关的存档列表。  
它不等于“个人游戏库”。

### 3.6 其它 `···`

当前至少承载：

- 当前运行配置查看
- 当前模型切换入口

如果剪辑分享在本轮进入 MVP，也放这里，不放汉堡菜单。

---

## 四、中段消息流

### 基础职责

- 显示用户消息
- 显示 AI 消息
- 显示流式输出
- 显示回合选项

### 当前不承担

- 立绘
- 背景层
- VN 对话框
- 群聊复杂角色切换

### 渲染规则

| 类型 | 样式 |
|------|------|
| `user` | 右对齐 |
| `assistant` | 左对齐 |
| `streaming` | 左对齐，持续追加 |
| `system` | 低强调度提示 |

### 选项按钮

如果后端返回选项，则显示在最近一条 AI 回复下方。  
如果没有返回选项，则不额外制造假按钮。

---

## 五、底端输入区

从左到右固定为：

1. **汉堡菜单**
2. **输入框**
3. **发送按钮**

### 5.1 汉堡菜单

当前 MVP 只要求两个功能：

- `AI 帮答`
- `重新生成`

其他功能不再提前堆进第一版。

### 5.2 输入框

输入框占位文本统一来自：

- `ui_config.input_placeholder`

这样可以做到：

- 不同游戏给出不同输入引导
- 文案精准，不靠前端硬编码猜测

### 5.3 发送按钮

- 空闲时：发送
- 生成中：停止

保持唯一主操作，不再并排增加更多按钮。

---

## 六、模型配置位置

### 全局维护

全局模型配置维护在：

- `MyLibraryPage`

维护内容包括：

- `base_url`
- `api_key`
- `model_label`

### 单局切换

单局只在 TopBar 的 `···` 中切换：

- 当前 `model_profile`
- 当前 `engine_mode`

不在游玩页直接编辑 URL / Key。

---

## 七、关于后端编排的约束

Text 页文档不再写：

- `One-shot`
- “每回合只调用一次 LLM”
- 依赖 narrator/director/verifier 次序的前端说法

前端只假设：

- 有稳定回合接口
- 有稳定流式输出
- 有稳定设定与状态查看入口

---

## 八、当前不做

以下功能从 Text 页当前文档中删除：

- Text 游戏绕过详情页直接进入聊天
- Swipe 页码交互细则
- Text 页内评论区入口
- 过度复杂的 ST 全量功能映射
- 与目录页和详情页重复的内容

这些内容不是永久废弃，只是**不属于当前 Text 页专项文档**。
