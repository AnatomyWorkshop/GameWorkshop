## Bug 1（阻断）：session.id 不存在 → 导航到 /play/undefined

**现象**：点击游戏卡片后跳转到 `/play/undefined`，PlayPage 无法加载。

**根因**：后端 `POST /api/play/sessions` 返回 `{session_id: "..."}` 而非完整 Session 对象。
前端 `useCreateSession` 期望返回 `Session`（有 `.id` 字段），但实际拿到的是 `{session_id: "..."}` 对象，`.id` 为 `undefined`。

**位置**：
- `src/api/sessions.ts` — `create()` 返回类型声明为 `Session`，实际后端返回 `{session_id: string}`
- `src/pages/game-list/GameListPage.tsx:17` — `nav('/play/${session.id}', ...)`

**修复**：`sessionsApi.create` 返回类型改为 `{session_id: string}`，导航时用 `session.session_id`。

## Bug 2（阻断）：`gamesApi.worldbook` 路由 404

**现象**：调用 `GET /api/play/games/:id/worldbook-entries` 返回 404。

**根因**：后端路由 `/games/:id/worldbook-entries` 与 `/games/:slug` 冲突，已改为 `/games/worldbook/:id`，但前端 `games.ts` 未同步。

**位置**：`src/api/games.ts:15`

**修复**：路径改为 `/play/games/worldbook/${id}`。（注：该 API 目前未在 MVP 页面中使用，不阻断主流程，但需修复避免报错。）

## Bug 3（轻微）：SSE `done` 事件未处理，流不会正常结束

**现象**：SSE 流结束后 `streaming` 状态可能不清除（取决于 `fetchEventSource` 的 `onclose` 行为）。

**根因**：后端在 `meta` 事件后还会发一个 `event:done\ndata:\n` 事件，前端 `sse.ts` 的 `onmessage` 只处理 `token`/`meta`/`error`，`done` 事件被忽略。`fetchEventSource` 在连接正常关闭时会调用 `onclose`，但未设置该回调。

**位置**：`src/api/sse.ts:17-22`

**修复**：添加 `onclose` 回调调用 `onDone`（若尚未调用），或在 `onmessage` 中处理 `done` 事件。实际上 `meta` 事件已经携带完整数据，`done` 只是信号，加 `onclose` 兜底即可。

## Bug 4（轻微）：`useFloors` 返回的 `messages` 内容乱码

**现象**：Python 打印 floors 时 content 显示乱码（`\udcac` 等 surrogate 字符）。

**根因**：Python 终端编码问题，实际数据是正确的 UTF-8，前端渲染不受影响。非 bug。

---

## Bug 5（阻断）：`@/` 路径别名未配置 → Vite 无法解析 CSS

**现象**：启动前端报错：
```
[plugin:vite:import-analysis] Failed to resolve import "@/styles/globals.css" from "src/main.tsx"
```

**根因**：`vite.config.ts` 没有配置 `resolve.alias`，`@/` 别名无效。文件本身存在（`src/styles/globals.css`），但 Vite 不知道 `@` 指向 `src/`。

**位置**：`vite.config.ts` — 缺少 `resolve.alias`

**修复**：在 vite.config.ts 中添加：
```ts
import path from 'path'
resolve: { alias: { '@': path.resolve(__dirname, 'src') } }
```
同时 `tsconfig.json` 需要对应的 `paths` 配置（通常已有）。

**状态**：✅ 已修复
